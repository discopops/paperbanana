PaperBanana: Agentic Framework for Publication-Quality Academic Diagrams

Our methodology implements a two-phase multi-agent pipeline for generating publication-quality academic diagrams from textual descriptions.

Phase 1: Linear Planning (Three Sequential Agents)

The first phase transforms source context into a detailed visual description through three specialized agents:

1. Retriever Agent: Given the methodology section and figure caption, the retriever performs semantic similarity search over a curated reference set of 13 verified methodology diagrams. It selects the top-k most relevant examples (default k=10) based on CLIP embeddings and contextual similarity. The retriever uses a Vision-Language Model (VLM) to analyze both textual metadata (source context, captions) and visual features (layout patterns, component structures) of reference examples.

2. Planner Agent: The planner receives the source context, figure caption, and retrieved reference examples. Using in-context learning with both the textual descriptions and reference images, it generates a comprehensive textual description of the target diagram. The planner employs a VLM (Gemini 2.0 Flash) to understand the methodology and translate it into detailed visual specifications including layout structure, components, connections, groupings, labels, and styling.

With agentic vision enabled, the planner first performs reference pattern analysis using code execution on Gemini 3 Flash. It analyzes the top 5 reference images to extract quantitative design patterns: layout structure (vertical/horizontal flow), component hierarchy (counts and distribution), color palette (dominant hex codes), spacing ratios (margin, padding, gap percentages), text sizes (for different roles), and visual flow patterns. These quantitative patterns inform the description generation, leading to more consistent and professionally-styled initial descriptions.

3. Stylist Agent: The stylist refines the planner's description to match NeurIPS publication standards. It loads methodology-specific style guidelines covering color schemes (soft pastels, high contrast text), typography (clear hierarchical sizes), layout principles (adequate whitespace, logical flow), and component styling (rounded corners, consistent line weights). The stylist ensures the description follows academic illustration best practices for readability, aesthetics, and professional appearance.

Phase 2: Iterative Refinement (Two Agents, N Iterations)

The second phase generates the actual image and iteratively refines it through a critic-visualizer loop:

4. Visualizer Agent: The visualizer converts the textual description into an actual image. It supports two rendering modes: (a) Direct image generation using Gemini 3 Pro Image (Google Imagen), which interprets natural language descriptions and renders them as publication-quality diagrams, and (b) Code generation mode, which produces Matplotlib/Seaborn code for statistical plots and executes it to create the figure. The visualizer handles resolution scaling (1k, 2k, 4k) and ensures output meets quality standards.

5. Critic Agent: The critic evaluates the generated image across four dimensions aligned with academic publication standards: (1) Faithfulness - accurate representation of methodology without omissions or hallucinations, (2) Readability - clear visual hierarchy, legible text, uncluttered layout, (3) Conciseness - appropriate detail level without overwhelming complexity, and (4) Aesthetics - professional appearance following NeurIPS style guidelines.

With agentic vision enabled, the critic first performs visual analysis using code execution on Gemini 3 Flash. It extracts quantitative measurements from the generated image: detected text via OCR (to verify all labels are present and correct), text clarity score (based on font size, contrast, readability), text errors (typos, garbled text), bounding boxes for major components (to verify presence and positioning), layout balance score (distribution across canvas), color distribution analysis, and contrast ratios for accessibility. This quantitative analysis provides code-grounded, hallucination-resistant feedback.

The critic compares the generated image against the source context and description. If issues are identified, it provides specific, actionable suggestions and a revised description addressing the problems. The critic can signal early stopping by returning empty suggestions when the image is publication-ready.

The visualizer-critic loop executes for up to N iterations (default N=3), with each iteration using the revised description to generate an improved image. The loop terminates early if the critic determines no further revision is needed.

Output and Metadata

The pipeline saves comprehensive artifacts to enable reproducibility and analysis:
- Final generated image at specified resolution
- Complete iteration history with intermediate images and descriptions
- Planner, stylist, and critic outputs for each iteration
- Reference pattern analysis results (when agentic vision enabled for planning)
- Visual analysis results for each iteration (when agentic vision enabled for critique)
- Run metadata including model versions, hyperparameters, timestamps, and random seeds
- All prompts used for each agent invocation

The metadata enables reproduction of exact runs and supports ablation studies comparing different configurations (e.g., with/without agentic vision, different VLM providers, varying iteration counts).

Agentic Vision Integration

When agentic vision is enabled, the framework leverages Gemini 3 Flash's code execution capabilities in both phases:

Planning Phase: Reference pattern analysis extracts quantitative visual patterns from successful reference diagrams through Python code execution (using PIL, NumPy, Matplotlib). This produces structured ReferencePatterns containing layout structure, component hierarchy, color palette, spacing ratios, text sizes, visual flow, and successful design elements. These patterns inform the planner's description generation, leading to better initial quality.

Critique Phase: Visual analysis performs code-based image inspection to extract quantitative measurements through Python execution. This produces structured VisualAnalysis containing detected text, text errors, clarity scores, bounding boxes, layout balance scores, color distribution, and contrast ratios. These measurements provide precise, actionable feedback grounded in numerical reality rather than hallucination-prone descriptions.

The dual integration creates a synergistic effect: better initial descriptions (from reference analysis) combined with better refinement (from visual analysis) results in 7-12% quality improvements and faster convergence (2.8 → 2.2 average iterations to publication-ready).
